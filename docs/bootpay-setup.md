# 부트페이 구독 결제 — 관리자 설정 체크리스트

`RC_PM_NOT_FOUND`(결제수단 정보를 찾지 못했습니다)가 나오면 아래를 순서대로 확인하세요.

---

## 1. PG 설정 (부트페이 관리자)

**경로:** [admin.bootpay.co.kr](https://admin.bootpay.co.kr) → **PG 설정** 탭

- [ ] **나이스페이먼츠**가 **사용중**으로 선택되어 있는지
- [ ] **결제수단 선택하기**에서 **카드정기결제**가 **실결제**로 켜져 있는지 (토글 ON)
- [ ] 나이스페이 연동을 위해 부트페이에서 요구하는 **가맹점 ID / 비밀키** 등을 나이스페이에서 발급받아 부트페이 PG 설정란에 입력했는지

> 나이스페이와의 계약·연동이 완료되어야 카드정기결제가 동작합니다. 부트페이 쪽에 나이스 가맹점 정보가 없으면 결제수단을 찾을 수 없습니다.

---

## 2. 개발자 설정

**경로:** **개발자 설정** 탭

- [ ] 이 프로젝트에서 쓰는 **JavaScript WEB** 키가 구독 페이지의 `NEXT_PUBLIC_BOOTPAY_APPLICATION_ID`와 같은지
- [ ] **테스트 모드**를 쓰는 경우, 해당 프로젝트에서 테스트 결제가 허용되는지
- [ ] **도메인 허용** 목록에 현재 접속 주소가 들어 있는지 (예: `localhost:3001` 또는 배포 도메인)

---

## 3. 나이스페이 가맹점 연동

- [ ] **나이스페이**에 가맹점으로 등록되어 있고, **정기결제(빌링)** 서비스가 개통되었는지
- [ ] 부트페이 PG 설정에 나이스에서 받은 **MID, 비밀키** 등이 올바르게 입력되었는지

---

## 4. 코드에서 사용 중인 값

- **PG:** `nicepay` (나이스페이먼츠)
- **method:** `card_rebill` (카드 정기결제/빌링)

관리자에서 **카드정기결제**만 켜져 있고, 나이스 가맹점 연동이 되어 있다면 위 설정으로 결제창이 뜨는 것이 정상입니다.  
계속 오류가 나면 부트페이 고객센터(채널 등)에 **프로젝트 ID + RC_PM_NOT_FOUND + 나이스 카드정기결제**라고 문의하면 원인 파악에 도움이 됩니다.
