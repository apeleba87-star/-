# 업계 소식 메뉴 노출 방식 제안

리포트가 쌓이는 "업계 소식" 메뉴를 눌렀을 때, 사용자에게 어떻게 보여주면 좋을지 옵션 정리입니다.

---

## 현재 구현 (기본)

- **경로**: `/news` (업계 소식 메뉴 클릭 시)
- **내용**: `source_type = auto_tender_daily` 인 발행된 글만 최신순 목록 (카드 형태, 제목·요약·날짜)
- **상세**: 카드 클릭 시 기존 `/posts/[id]` 또는 `/posts/[slug]` 로 이동 (대시보드 리포트 뷰)

---

## 사용자 친화적 노출 옵션

### 1. 목록 → 상세 (현재 방식)

- **진입**: 업계 소식 → **카드 목록** (최신순)
- **카드**: 제목, 요약 1~2줄, 날짜, "입찰 리포트" 뱃지
- **클릭**: 해당 글 상세(리포트 대시보드)로 이동

**장점**: 구조 단순, 스크롤로 과거 소식 탐색 쉬움.  
**단점**: “오늘 것만 빠르게 보기” 같은 강조는 약함.

---

### 2. 최신 1건 하이라이트 + 목록

- **첫 화면**: "오늘/최신 리포트" 한 건을 상단에 크게 (제목 + 요약 + 건수 등 요약 수치 + [보기] 버튼)
- **아래**: "이전 소식" 목록 (카드 리스트, 현재와 동일)

**장점**: 가장 최신 리포트를 강조, “오늘 입찰 몇 건?”을 한눈에 보여줄 수 있음.  
**단점**: “최신 1건” 정의 필요 (당일 리포트만? 없으면 전날? 등).

---

### 3. 타임라인 / 연·월 구분

- **진입**: 업계 소식 → **연도·월** 접이식 또는 타임라인
- 예: "2026년 3월" 펼치면 → "3/12 입찰 34건", "3/11 입찰 47건" …
- 클릭 시 기존 상세로 이동

**장점**: 날짜로 찾기 좋음, 오래된 소식 탐색에 유리.  
**단점**: UI·데이터 구조가 목록형보다 조금 무거움.

---

### 4. 필터/탭 (나중에 확장용)

- **탭 예시**: "입찰 리포트" | "업계 칼럼" (수동 글)
- `source_type` 또는 카테고리로 구분해, "입찰 리포트"만 / "전체 업계 소식"만 보기

**장점**: 나중에 수동 칼럼·이슈 등이 늘어나도 같은 메뉴에서 구분 가능.  
**단점**: 지금은 자동 리포트만 있어도 되므로, 콘텐츠가 늘어날 때 도입해도 됨.

---

## 추천

- **당장**: **1번(목록 → 상세)** 유지. 이미 구현했고, 사용자도 이해하기 쉬움.
- **다음 단계**: "오늘/최신 리포트"를 강조하고 싶다면 **2번**처럼 상단에 최신 1건 하이라이트 블록만 추가.
- **소식이 많이 쌓이면**: **3번** 타임라인/연·월로 넘어가면 탐색이 편해짐.

정리하면, **지금은 목록(카드) → 상세(리포트)** 구조로 두고, 필요해지면 "최신 1건 하이라이트"나 "연·월 타임라인"을 단계적으로 붙이는 방식이 사용자에게도 편합니다.
