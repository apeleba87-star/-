# 클린아이덱스 — 입주수요지수 · 지역 탐험 (기획안 v3.0)

> v2.0(영업 레이더) + v2.1(API 현실 반영) + 입주수요지수 1차안 + 체류·탐험 전략을 통합한 **실행 가능** 기획안  
> 최종 수정: 2026-05

---

## 1. 한 줄 정의

**청소·이사 업체 사장이 「이번 달 어디에 광고비를 쓸지」 스스로 확신할 수 있게,  
지역별 단서·비교·과거 적중을 탐험하는 무료 도구**

| 우리는 | 우리가 아닌 것 |
|--------|----------------|
| 청소업 **영업·광고 의사결정** 도구 | 부동산 투자·시세 플랫폼 |
| **지역 탐험** 서비스 (확신을 쌓는 구조) | 「강서구 좋음」 한 줄 예측 서비스 |
| 키워드마스터 대비 **입주·이사 수요 맥락** | 검색량 표만 보여주는 도구 |

---

## 2. 핵심 문제 · 원칙

### 2.1 문제

현재처럼 **「강서구 · 입주수요지수 132 · 상승 예상」** 만 보여주면  
사용자는 *아 그렇구나* 하고 **3초 만에 이탈**한다.

### 2.2 사용자가 원하는 것

| 원하는 것 | 원하지 않는 것 |
|-----------|----------------|
| **확신** (왜? 비교하면? 맞았나?) | **답 한 줄** (132면 끝) |
| 광고비 배분 근거 | raw 데이터만 |
| 남의 지역·변화·순위 | 숫자 하나 |

### 2.3 UX 원칙

```
요약(지수) → 근거 → 드릴다운 → 지역 비교 → 과거 적중 → (선택) 한 줄 참고
```

- **답을 주지 말고 단서를 준다** — 클릭할수록 깊어지게
- **탐험 루프**: 강서 → 양천 → 마포 → … (키워드마스터의 「다른 키워드 클릭」= 「다른 구 클릭」)
- 탐험 **끝**에만 참고용 한 줄: *「이번 달만 보면 강서·양천 신호가 큽니다」* (강제 X)

### 2.4 표현 원칙 (면책·신뢰)

- **「상승 예상」** 대신 **「신호 강함 / 관망 / 약함」** (예측 CS 리스크 완화)
- **구별** vs **전국** 데이터는 UI에 **항상 라벨** 표기
- 모든 페이지 하단 공통 면책:

> 국토부 실거래·행안부 인구이동(선택)·네이버 검색 트렌드 집계 기준입니다.  
> 검색 수치는 상대값이며 실제 건수가 아닙니다. 신고·집계 지연(통상 1~2개월)이 있을 수 있습니다. 참고용입니다.

## 3. 기본 가설 (제품 스토리)

### 3.1 고객 행동 순서

```
부동산 계약 (매매·전월세)
  → 포장이사 검색 (선행)
  → 입주청소 검색
  → 실제 입주
  → 청소 문의
```

### 3.2 선행 가설 (검증 중)

**포장이사 검색(전국)이 입주청소 검색(전국)보다 약 1개월 선행**할 가능성  
— 강서구 관찰 사례(2024·2025)를 출발점으로, **5구 × 36개월** 백테스트로 적중 아카이브 축적

### 3.3 포지셔닝 비교

| | 키워드마스터 | 클린아이덱스 |
|--|-------------|-------------|
| 질문 | 이 키워드 검색량은? | **이번 달 어디에 광고비?** |
| 체류 | 직접 분석·표 탐색 | **구 탐험·비교·적중** |
| 차별 | 검색량 조회 | **입주수요 맥락 + 거래 검증 + 적중 기록** |

## 4. 데이터 — 사용 가능한 것만

### 4.1 확정 소스

| 지표 | API·인프라 | 공간 | 시간 | MVP |
|------|------------|------|------|-----|
| 전월세 거래 건수 | 국토부 RTMS 아파트 전월세 | **구별** (서울 25구) | 월별 | ✅ |
| 매매 거래 건수 | 국토부 RTMS 아파트 매매 | **구별** | 월별 | ✅ |
| 순전입 | 행안부 지역별 인구이동 (구 합산) | **구별** | 월별 | ✅ (v3.1) |
| 포장이사·입주청소 등 | 네이버 데이터랩 (`lib/naver/*`, `/marketing-report`) | **전국만** | 일/주/월 | ✅ |
| 적중 아카이브 | 내부 백테스트 → DB | 전국·구(거래) | 월별 | ✅ (검증된 것만 노출) |
| 관련 입찰 | 기존 `tenders` | 시도·기관 텍스트 | 실시간 | ✅ 보조 링크 |

### 4.2 v1.0 제외

| 항목 | 이유 |
|------|------|
| 구별 포장이사·입주청소 검색량 | 검색광고 API도 **전국·최근 30일 추정**만 (구별 없음) |
| 구별 검색 **지수** | 데이터랩은 **상대 지수**만, 지역 파라미터 없음 |
| 네이버 부동산·크롤링 | 법적·운영 리스크 |
| 신축·분양·실시간 주간 지수 | 안정적 공공 API 없음 |
| 전국 229시군구 | 서울 25구 MVP 후 확장 |

### 4.3 화면에 보이는 4지표 vs 실제

**사용자에게는 4개 근거를 모두 보여준다.**  
**내부 계산·라벨은 데이터 성격에 맞게 구분한다.**

| 근거 (UI) | 실제 범위 | 역할 |
|-----------|-----------|------|
| 주택매매 지수 | **해당 구** | RTMS 매매 건수·전월 대비 |
| 전월세 거래지수 | **해당 구** | RTMS 전월세 건수·전월 대비 |
| 포장이사 지수(검색량) | **전국** | 검색광고 **검색량(전월·30일)** + 데이터랩 **지수(전월%)** · 일간 **지수(전일)** 는 펄스 |
| 입주청소 지수(검색량) | **전국** | 동일 |

> 구 상세에서 검색 2줄 옆에 `(전국)` 배지 필수.  
> 「강서구 포장이사 22,000」처럼 **구별 절대값은 v1.0 금지**.

## 5. 입주수요지수

### 5.1 이름 · 스케일

- **입주수요지수** (영업 활성 지수 대신 통일)
- **100 = 기준** (서울 25구 최근 12개월 중앙값 또는 전월 대비 정규화)
- 예: **132** → 「평소 대비 활발」 정도로 해석 가능하게 툴팁 제공

### 5.2 가중치 (가설 v0 — 백테스트 후 조정)

| 구성 | 가중 | 데이터 | 비고 |
|------|------|--------|------|
| 포장이사 검색 (전국) | **40%** | datalab MoM | 구별 불가 → 전국 신호 |
| 전월세 거래 (구) | **35%** | RTMS | |
| 매매 거래 (구) | **15%** | RTMS | |
| 입주청소 검색 (전국) | **10%** | datalab | 후행; v0.1 설계 50/25/15/10에서 전국 반영으로 조정 |
| 순전입 (구) | (v3.1 +5~10%) | 행안부 | v3.0은 근거 카드에만, 지수 합산은 Phase 1.5 |

**내부 문서:** 가중치는 **「가설 v0」** — `/demand/hits` 적중률로 v1 재조정.

### 5.3 신호 라벨 (예측 대체)

| 라벨 | 조건 (예시) |
|------|-------------|
| 🟢 **신호 강함** | 구 거래 MoM 상위 + 전국 포장이사 MoM > 0 |
| 🟡 **관망** | 지표 엇갈림 |
| 🔴 **신호 약함** | 거래·키워드 동반 하락 |

### 5.4 근거 3+1 (항상 노출)

구 카드·상세 상단에 **접지 않고** 4줄 요약:

1. 주택매매 지수 (구) 건수 · ▲/▼ %
2. 전월세 거래지수 (구) 건수 · ▲/▼ %
3. 포장이사 지수(검색량) ▲/▼ %
4. 입주청소 지수(검색량) ▲/▼ %

각 줄 **클릭 → 드릴다운** (전략 1).

### 7.4 구 탐험 `/demand/region/[gu]`

**레이어 0 — 요약 (스크롤 없이)**

```
강서구 · 입주수요지수 132 · 🟢 신호 강함
서울 순위 1위 (지난달 5위) · 2025년 4월 기준
```

**레이어 1 — 근거 4줄 (항상 보임)**  
각 줄 ▶ 클릭

**레이어 2 — 드릴다운 (클릭 시)**

| 클릭 | 내용 |
|------|------|
| 포장이사 (전국) | 최근 3~6개월 차트 + MoM + 「서울 구 거래와 함께 볼 때」 해석 1줄 |
| 전월세 (구) | 월별 건수 차트 + **25구 순위** + 지난달 순위 |
| 매매 (구) | 동일 |
| 입주청소 (전국) | 차트 + 선행 관계 설명(가설) |

**레이어 3 — 탐험 링크**

- **양천구·마포구와 비교** → `/demand/compare?a=강서구&b=양천구&c=마포구`
- **비슷한 패턴 구** 3곳
- **순위 이웃** (2위 양천, 3위 마포)
- **적중** 이 구/전국 관련 카드 1~2개

**레이어 4 — 보조 (체류·클린아이덱스 허브)**

- 관련 청소 입찰 N건 → `/tenders?…`
- 검색 트렌드 전체 → `/marketing-report`
- 견적 참고 → `/estimate`

**레이어 5 — 참고 한 줄 (푸터 위)**

> 이번 달 데이터만 보면 강서·양천 쪽 신호가 가장 큽니다. 채널·예산은 업체마다 다릅니다.

**광고:** 구 하단 1슬롯 (스카이·장비·약품 직접광고)

### 7.5 구 VS 구 `/demand/compare` ⭐ 체류 핵심

- 기본 프리셋: `강서 vs 양천 vs 마포`
- URL 공유 가능
- 표:

| 지표 | 강서 | 양천 | 마포 |
|------|------|------|------|
| 입주수요지수 | 132 | 108 | 105 |
| 전월세 (구) | +23% | +8% | +5% |
| 매매 (구) | +18% | +5% | +3% |
| 포장이사 (전국) | 동일값 + 각 구 해석 | | |
| 입주청소 (전국) | 동일 | | |

- **「+ 구 추가」** (최대 4구)
- **내 영업 구역** 불러오기
- 하단: 채널 참고 ★ (규칙 v0.1, `?` 도움말)

## 15. v3.0 한 줄 요약

**입주수요지수는 입구일 뿐이고,  
제품은 「왜 · 비교 · 변화 · 맞았나」를 구간 구간 탐험하게 만드는 지역 탐험 서비스다.  
데이터는 구별 거래 + 전국 검색 + 적중 기록만 쓰고, 없는 구별 검색량은 쓰지 않는다.**
