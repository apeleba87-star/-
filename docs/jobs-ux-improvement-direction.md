# 인력 구인 UX 개선 방향 — 현장 거래 수준으로 정리

현장 거래(글쓰기·목록·상세)는 단계별 흐름·일관된 스타일·자동 제목·금액 포맷 등으로 정리되어 있습니다. 인력 구인도 같은 톤으로 맞추면 인지 부하가 줄고, 작성·탐색 경험이 통일됩니다.

---

## 1. 현장 거래 vs 인력 구인 (현재 비교)

| 구분 | 현장 거래 (listings) | 인력 구인 (jobs) 현재 |
|------|----------------------|------------------------|
| **글쓰기 구조** | 5단계 StepSection (어떤 현장 → 어디 → 규모 → 금액 → 설명) | 2개 블록 (현장 공통 정보 + 모집 포지션) |
| **제목** | 자동 제안(지역+업무+규모+유형), 수정 가능 | 사용자가 처음부터 직접 입력 |
| **입력 순서** | “생각의 흐름”에 맞춤 (유형→지역→규모→금액→설명) | 제목·지역·주소·일시·연락처·설명·확정자 안내가 한 섹션에 혼재 |
| **금액 표기** | 190,000원 형식(콤마), formatAmountWon 등 공통 유틸 | 천 단위 콤마만, 라벨/단위 표기가 폼마다 상이할 수 있음 |
| **카테고리/업무** | 버튼 그룹(소개/매매/도급 → 정기/일회성 → 사무실·계단 등) | JOB_TYPE_PRESETS 버튼 + 기타 텍스트 (한 단계) |
| **시각적 구분** | Step 번호 + 제목이 있는 카드형 섹션 | glassCard 섹션 2개, 단계 번호 없음 |
| **선택/필수** | 단계별로 “필수/선택” 문구·placeholder 명확 | 일부만 툴팁/문구 있음 |

## 2. 개선 방향 제안

### 2.1 글쓰기: 단계별 흐름으로 재구성 (현장 거래처럼)

**현재**  
- 한 화면에 “현장 공통 정보” + “모집 포지션” 두 덩어리  
- 제목을 맨 처음에 비어 있는 상태로 입력하게 함  

**개선**  
- **Step 1 — 어떤 현장인가?**  
  - “어떤 작업을 구하는지” 먼저 선택  
  - 작업 종류: 기존 JOB_TYPE_PRESETS 버튼 그룹 유지하되, 한 줄 요약 문구 추가  
  - (선택) 대분류가 있다면 “일회성/정기” 등 그룹이 있으면 1단계에서 선택  

- **Step 2 — 어디·언제인가?**  
  - 지역(시·구), 작업일, 작업 시간대(오전/오후/종일 프리셋 + 직접 입력)  
  - “이 작업 일시를 모든 포지션에 적용” 체크는 여기 유지  

- **Step 3 — 모집 포지션**  
  - 포지션 추가/삭제, 포지션별: 작업 종류(Step1과 연동 가능), 숙련도, 인원, 금액(190,000원 형식), 지급 단위  
  - 반당 시 시간대, 포지션별 시간 등은 Step 3 안에만 두어 “모집 내용”에 집중  

- **Step 4 — 연락처·상세·확정자 안내**  
  - 연락처(필수), 현장 간편 설명(본문), 상세 주소  
  - “확정된 지원자에게만 공개” 블록(출입 방법, 주차, 주의사항)을 한 덩어리로 묶고, 접기/펼치기 가능하면 더 정리됨  

- **Step 5 — 제목 확인·등록**  
  - **제목 자동 제안**: 지역 + 대표 작업 종류 + 작업일(또는 “N개 포지션”) + “구인” 등  
  - 사용자가 수정 가능한 한 줄 입력  
  - 등록 버튼은 이 단계에  

이렇게 하면 “어떤 작업 → 어디·언제 → 몇 명·얼마 → 연락처·설명 → 제목 확인” 순서로, 현장 거래의 “어떤 현장 → 어디 → 규모 → 금액 → 설명”과 비슷한 리듬이 됩니다.

### 2.2 제목 자동 제안

- 입력값 기반으로 **자동 제안** 한 줄 생성  
  - 예: `서울 강남 · 유리청소·준공청소 · 3/15 · 구인` 또는 `서울 강남 3포지션 3/15 구인`  
- 제목 필드는 “자동 생성된 제목입니다. 필요하면 수정하세요” 같은 안내와 함께 기본값으로 채움  
- 수정 가능하게 두어, 구인글 특성(특별 강조 문구 등)은 그대로 반영 가능  

### 2.3 금액·숫자 표기 통일

- 현장 거래와 동일하게 **190,000원** 형식(콤마 + “원” 표기)  
- `formatAmountDisplay` / `parseAmountInput` / `formatAmountWon` 등 공통 유틸을 인력 구인 폼·카드·상세에도 적용  
- 지급 단위(일당/반당/시급)는 기존처럼 유지하되, “190,000원 (일당)”처럼 단위를 같이 노출하면 일관성 있음  

### 2.4 시각적 일관성 (StepSection 스타일)

- 현장 거래와 동일한 **StepSection** 패턴 도입  
  - 상단: 원형 번호 + 단계 제목  
  - 하단: 흰색 카드형 배경 + 테두리  
- 인력 구인 전용 색/강조만 살짝 다르게 해도 되고(예: 파란 계열 유지), 레이아웃·간격·타이포는 동일하게 맞추면 “같은 앱” 느낌이 남  

### 2.5 목록·필터·카드

- **목록 툴바**: 정렬·필터(지역, 작업종류, 숙련도, 작업일 등)는 그대로 두고, 버튼/드롭다운 스타일만 현장 거래 목록과 비슷하게 (둥근 버튼, 일관된 padding 등)  
- **카드**:  
  - 지역·작업일·포지션 요약·급여·지원자 수 등 정보 계층을 명확히  
  - “오늘 현장”/“내일 현장” 같은 뱃지는 유지하되, 현장 거래 카드와 비슷한 여백·폰트 크기로 통일  
- **상세 페이지**:  
  - “현장 정보 / 모집 포지션 / 지원하기” 등 블록을 번호나 제목으로 나누어, 글쓰기 단계와 대응되면 이해하기 쉬움  

### 2.6 선택 입력 정리

- “현장 상세 주소”는 Step 4에 두고, “확정된 지원자에게만 공개” 문구를 바로 옆에  
- “현장 안내(출입·주차·주의사항)”는 하나의 접을 수 있는 블록으로 묶고, 기본은 접힌 상태로 두어도 됨  
- 필수/선택을 라벨에 “*” 또는 “(선택)”으로 명시해 두면 현장 거래와 동일한 톤

## 3. 우선 적용하면 좋은 순서

1. **글쓰기 단계화**  
   Step 1~5 구조로 나누고, 제목을 마지막 단계로 이동 + 자동 제안 적용  

2. **금액 포맷 통일**  
   인력 구인 폼·카드·상세에 190,000원 형식 및 공통 유틸 적용  

3. **StepSection UI 적용**  
   글쓰기 화면에 현장 거래와 동일한 StepSection 컴포넌트(또는 동일 스타일) 적용  

4. **목록·카드 스타일 정리**  
   카드·툴바를 현장 거래 목록과 비주얼만 맞춤  

5. **상세 페이지 블록 정리**  
   정보를 “현장·일시 / 모집 포지션 / 연락처·안내” 등으로 나누어 가독성 개선  

---

## 4. 정리

- **인력 구인**을 **현장 거래처럼 깔끔하게** 만들려면,  
  - 글쓰기를 **단계별(어떤 작업 → 어디·언제 → 포지션 → 연락처·상세 → 제목)** 로 재구성하고  
  - **제목 자동 제안**, **금액 190,000원 형식**, **StepSection 스타일**을 맞추면 됩니다.  
- 목록·상세는 구조를 크게 바꾸지 않고도, **스타일·정보 계층**만 통일하면 같은 품질 체감을 줄 수 있습니다.  
- 구현 시에는 `JobPostForm`을 Step 단위로 쪼개고, 제목은 `useMemo`로 자동 제안한 뒤 마지막 단계에서 한 번에 확인·수정하는 방식이 현장 거래와도 가장 비슷합니다.
