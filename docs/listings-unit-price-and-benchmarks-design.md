# 현장거래 평균 단가 — 실제 데이터 기반 시장 흐름 장치

> **정책: 관행 안내 없음**  
> 10%, 1~6배, 80% 등 **업계 관행 안내 문구는 사용하지 않는다.** 평균·참고값은 **플랫폼에 쌓인 실제 등록 데이터만**으로 산출·노출한다.

> **보완점 적용 정리·구현 방안**: [listings-benchmarks-improvements-and-implementation.md](./listings-benchmarks-improvements-and-implementation.md)  
> (신뢰도 표시, 중앙값·fallback·이상치·용어·표현 등 수용 항목과 구현 순서 정리)

## 0. 핵심 원칙 (재정리)

### 0.1 “10% / 1~6배 / 80%”의 위치
- **정기소개 약 10%**, **매매 1~6배**, **도급 약 80%**는 **업계의 통상적인 암묵적 관행**이다.
- 이 수치는 **우리 시스템의 기준이나 고정값이 아니다.**  
  → 사용자에게 “참고용 관행”으로 안내할 수는 있지만, **평균 단가를 이걸로 계산하거나 보여주지 않는다.**

### 0.2 평균 단가는 “우리 플랫폼 실제 데이터”만 사용
- **평균 단가**는 **우리 시스템에 사용자들이 올린 매매글·도급글·소개글에서 입력한 금액/수수료**만 모아서 집계한 값이다.
- 즉, **실제 데이터를 바탕으로 평균을 구성**하고, 그 결과를 “시장 평균”으로 보여준다.
- 데이터가 쌓일수록 평균이 더 안정되고, 지역·업무·조건별로 나누면 더 의미 있는 “시장 흐름”이 된다.

### 0.2 평균 단가의 역할 — “글 올릴 때 시장 흐름 읽기”
- **목적**: 사용자가 **매매글·도급글·소개글을 올리면서 수수료·금액을 입력할 때**,  
  그 유형·조건에 맞는 **현재 시장 평균(평균 단가)**를 같이 보여줌으로써, **시장 전체의 흐름을 읽을 수 있게** 하는 것이다.
- **장치**:  
  - 글쓰기 화면에서 “소개비”, “월 수금”, “매매가”, “월 도급금” 등을 입력하는 **필드 옆 또는 아래**에  
    “이 지역·업무(·조건)의 **평균 소개비 / 평균 월 수금 / 평균 매매가 / 평균 월 도급금**은 약 〇〇만원입니다 (표본 N건)” 형태로 노출.  
  - 사용자는 자신이 입력하려는 값과 평균을 비교해서, 시장 흐름을 보고 금액을 결정할 수 있다.

**요약**  
- **기준**: 우리 플랫폼 **실제 등록 데이터**만으로 평균 단가 산출.  
- **역할**: 사용자가 **글을 올릴 때** 수수료·금액 입력 시 **평균 단가를 보여주는 장치**로, 시장 흐름을 읽게 함.

## 1. 업계 상황 정리 (입력 설계·집계 설계 참고용)

> 아래 10% / 1~6배 / 80%는 **업계 암묵적 관행**이다.  
> 우리는 이걸 “평균”으로 쓰지 않고, **어떤 항목을 사용자에게 입력받을지** 설계할 때만 참고한다.

### 1.1 정기청소 소개 (referral_regular)
- **관행**: 성사 금액의 약 10%를 소개비로 지급하는 경우가 많음.
- **흐름**: 견적 없이 소개만 → 소개받은 쪽이 현장 방문 후 견적 제시 → 계약 성사 시 소개비 지급.
- **우리가 할 일**: 사용자에게 **소개비(원)** 또는 **성사 금액 + 수수료율**을 입력받고, 그 **실제 입력값**으로 지역·업무별 **평균 소개비·평균 수수료율**을 계산해, 글쓰기 시 “현재 시장 평균”으로 보여준다.

### 1.2 정기청소 매매 (sale_regular)
- **매매가 결정 요인**: 어디(지역·업무), 몇 평, 주 몇 회, 난이도.
- **우리가 할 일**: 사용자에게 **월 수금**, **매매가**(또는 배수), **평수·주 회수·난이도**를 입력받고, 그 **실제 입력값**으로 조건별 **평균 월 수금·평균 매매가**를 계산해, 글쓰기 시 “현재 시장 평균”으로 보여준다.

### 1.3 현장 도급 (subcontract)
- **도급가 결정 요인**: 어디, 몇 평, 주 몇 회, 난이도.
- **우리가 할 일**: 사용자에게 **월 도급금**, **평수·주 회수·난이도**를 입력받고, 그 **실제 입력값**으로 조건별 **평균 월 도급금**을 계산해, 글쓰기 시 “현재 시장 평균”으로 보여준다.

## 2. “실제 데이터 → 평균 → 글쓰기 시 노출” 흐름

1. **데이터 수집**  
   사용자가 소개글·매매글·도급글을 올릴 때 입력한 **소개비, 성사 금액, 수수료율, 월 수금, 매매가, 월 도급금, 평수, 주 회수, 난이도** 등이 우리 DB에 쌓인다.

2. **평균 산출**  
   이 **실제 데이터만**으로, 거래 유형·지역·업무(·평수대·주회수·난이도)별로 평균을 계산한다.  
   (예: “서울·사무실청소·정기소개” 평균 소개비, “경기·일회성·도급” 평균 월 도급금 등.)

3. **글쓰기 시 노출**  
   사용자가 **새 글을 쓰면서** 소개비/월 수금/매매가/도급금 등을 입력하는 **그 순간**,  
   선택한 유형·지역·업무(·조건)에 해당하는 **현재 시장 평균(평균 단가)**를 같은 화면에 표시한다.  
   → 사용자는 “지금 시장이 얼마나 나오는지”를 보고 자신의 금액을 정할 수 있다. **(시장 전체 흐름을 읽는 장치)**

4. **10% / 1~6배 / 80%**  
   이 수치들은 **평균 계산에 쓰이지 않는다.**  
   필요하면 “보통 소개비는 성사 금액의 약 10%입니다”처럼 **안내 문구**로만 사용할 수 있다.

## 3. 현재 시스템과의 비교

### 3.1 현재 구조 (listings)
| 항목 | 현재 | 비고 |
|------|------|------|
| 거래 유형 | `listing_type`: referral_regular, sale_regular, subcontract 등 | 구분은 있음 |
| 금액 | `pay_amount` 1개 + `pay_unit` (일당/반당/시급) | **유형별 의미가 다름** |
| 환산 | `normalized_daily_wage` (일당 기준 통일) | 소개·매매·도급은 **월 단위**가 자연스러움 |
| 시장 평균 | `market_benchmarks`: 지역·카테고리·pay_unit별 `AVG(normalized_daily_wage)` | **listing_type 구분 없음**, 일당 중심 |
| 현장 조건 | 없음 | **평수, 주 회수, 난이도** 없음 |

### 3.2 유형별로 보면

**정기청소 소개**
- 현재: `pay_amount` = 소개비(원)로 두고, `pay_unit`은 일당 등으로만 쓸 수 있어 **성사 금액·수수료율**과 맞지 않음.
- 업계 평균을 내려면: **소개비** 또는 **성사 금액 + 수수료율**이 필요. 현재는 “소개비 1개”만 있어, 수수료율·성사 금액 평균을 낼 수 없음.

**정기청소 매매**
- 현재: `pay_amount`를 “매매가” 또는 “월 수금” 중 하나로만 쓸 수 있음. **월 수금**이 정해진 뒤 **배수(1~6)** 로 매매가가 정해지는 구조를 반영 못 함.
- 업계 평균을 내려면: **월 수금**, 그리고 선택적으로 **매매가**(또는 배수), **평수, 주 회수, 난이도**가 필요. 현재는 이 정보가 없어 “지역·업무·평수·주회수·난이도별 평균 월 수금/매매가”를 만들 수 없음.

**현장 도급**
- 현재: `pay_amount`를 “월 도급금”으로 쓸 수 있지만, `pay_unit`은 일당/반당/시급뿐이라 **월 단위**가 없음. 평수·주회수·난이도 없음.
- 업계 평균을 내려면: **월 도급금**(또는 평당·회당), **평수, 주 회수, 난이도**가 필요. 현재는 월 단위·조건 정보가 없음.

## 4. 제안: 유형별 단가·평균 체계

### 4.1 공통 원칙
- **유형별로 “단가” 정의를 나누고**, 입력 필드와 집계를 그에 맞춘다.
- **정기 매매·도급**은 “월 금액 + 현장 조건(평수, 주 회수, 난이도)”을 넣어, **조건별 업계 평균**을 낼 수 있게 한다.
- **정기 소개**는 “성사 금액 또는 소개비 + (선택) 수수료율”을 넣어, **소개비/수수료율 평균**을 낼 수 있게 한다.

### 4.2 데이터 구조 제안

**A. listings 확장 (유형 공통·선택)**

- **`listing_type`**: 유지.
- **기존**: `pay_amount`, `pay_unit`, `normalized_daily_wage` — 일회성·인력구인 등 기존 흐름 유지용.
- **추가 (선택 필드, 유형별로 사용)**  
  - `monthly_amount` (NUMERIC): 월 금액 (매매의 월 수금, 도급의 월 도급금 등).  
  - `deal_amount` (NUMERIC): 성사/매매 1회 금액 (소개는 성사 금액, 매매는 매매가).  
  - `fee_percent` (NUMERIC): 수수료율 (소개 시 10 등).  
  - `area_pyeong` (NUMERIC): 평수.  
  - `visits_per_week` (INT): 주 회수 (1~7 등).  
  - `difficulty` (TEXT): 난이도 (예: easy / normal / hard 또는 1~5).

**B. 단위 정리**

- **정기 소개**:  
  - 입력: `deal_amount`(성사 금액) 또는 `pay_amount`(소개비), 선택 `fee_percent`.  
  - 집계: 지역·업무별 평균 소개비, 평균 수수료율(입력된 경우만).
- **정기 매매**:  
  - 입력: `monthly_amount`(월 수금), 선택 `deal_amount`(매매가) 또는 배수, `area_pyeong`, `visits_per_week`, `difficulty`.  
  - 집계: (지역, 업무, 평수대, 주회수, 난이도)별 평균 월 수금, 평균 매매 배수.
- **현장 도급**:  
  - 입력: `monthly_amount`(월 도급금), `area_pyeong`, `visits_per_week`, `difficulty`.  
  - 집계: (지역, 업무, 평수대, 주회수, 난이도)별 평균 월 도급금(또는 평당·회당 환산).

**C. market_benchmarks 확장 (실제 데이터만 집계)**

- **기존**: region, category_main_id, category_sub_id, pay_unit, skill_level → average_pay, average_normalized_daily_wage. (일당 중심, 유형 무관)
- **추가 (유형별 집계)**  
  - `listing_type` (또는 별도 테이블/뷰): referral_regular, sale_regular, subcontract.  
  - **referral_regular**: 지역·업무별 평균 소개비, 평균 수수료율(%), 표본 수.  
  - **sale_regular**: 지역·업무·평수대·주회수·난이도별 평균 월 수금, 평균 매매가(또는 배수), 표본 수.  
  - **subcontract**: 지역·업무·평수대·주회수·난이도별 평균 월 도급금, 표본 수.

평수·주회수·난이도는 구간화(예: 평수 0~30, 31~50, … / 주 1~2회, 3~4회, …)해서 그룹 수를 제한할 수 있음.  
**모든 집계는 우리 플랫폼에 등록된 실제 사용자 입력값만 사용하며, 10% / 1~6배 / 80%는 사용하지 않는다.**

## 5. UX 제안 (유형별) — “글쓰기 시 평균 보여주기”

### 5.1 글쓰기 폼 (현장거래 등록) + 평균 단가 노출

- **거래 유형** 선택 시, 그에 맞는 입력 블록만 노출.

**정기청소 소개**
- **소개비(원)** * (현재 pay_amount에 대응하되, 라벨·placeholder만 “소개비”로).
- 선택: **성사 금액(원)** — 있으면 수수료율 자동 계산 표시(소개비÷성사금액×100).
- 선택: **수수료율(%)** — 직접 입력(10 등). 성사 금액과 둘 중 하나만 있어도 집계에 활용.
- 안내 문구: “보통 성사 금액의 약 10%를 소개비로 지급합니다.”

**정기청소 매매**
- **월 수금(원)** * (monthly_amount).
- **매매가(원)** 또는 **배수(1~6)** — 둘 중 하나 입력 시 다른 하나는 안내용으로만 표시 가능.
- **평수**, **주 회수**(1~7), **난이도**(쉬움/보통/어려움 등).
- **평균 노출**: "이 조건의 평균 월 수금 약 〇〇만원", "평균 매매가 약 〇〇만원 (표본 N건)" — **실제 등록 데이터만** 집계.
- 기존 제목, 지역, 업무 종류, 연락처, 상세 유지.

**현장 도급**
- **월 도급금(원)** * (monthly_amount).  
  안내: “원청 월 수금의 약 80% 수준으로 도급되는 경우가 많습니다.”
- **평수**, **주 회수**, **난이도**.
- 기존 제목, 지역, 업무 종류, 연락처, 상세 유지.

**일회성 소개**
- 현재처럼 `pay_amount` + `pay_unit`(일당 등) 유지해도 됨. 필요하면 “성사 금액/소개비” 선택 입력 추가.

### 5.2 상세·목록 노출

- **상세 페이지**:  
  - 유형별로 “이 건의 단가”를 한 줄로 표시 (소개비, 월 수금+매매가, 월 도급금 등).  
  - **시장 평균 비교**: 유형이 같고, 가능하면 조건(평수대, 주회수, 난이도)이 비슷한 건만 묶어 “업계 평균 소개비”, “업계 평균 월 수금”, “업계 평균 월 도급금” 등과 비교 (현재 MarketComparisonBox와 같은 패턴, 지표만 유형별로 분리).
- **목록**: 유형별로 “소개비”, “월 수금”, “월 도급금” 등 한 줄 요약 + (선택) “평균 대비 ±n%” 뱃지.

### 5.3 업계 평균 전용 화면 (선택)

- **필터**: 거래 유형, 지역, 업무 종류, (매매/도급 시) 평수대, 주 회수, 난이도.
- **결과**:  
  - 소개: 평균 소개비, 평균 수수료율(%), 표본 수.  
  - 매매: 평균 월 수금, 평균 매매가(또는 배수), 표본 수.  
  - 도급: 평균 월 도급금, 표본 수.  
- “지금 등록한 조건과 비슷한 업계 평균”을 보여주면, 등록 시 참고·비교용으로 쓸 수 있음.

---

## 6. 구현 단계 제안

1. **Phase 1 — 스키마·입력**  
   - listings에 `monthly_amount`, `deal_amount`, `fee_percent`, `area_pyeong`, `visits_per_week`, `difficulty` 추가(모두 nullable).  
   - 글쓰기 폼에서 `listing_type`에 따라 위 필드만 유형별로 노출.  
   - 기존 `pay_amount`/`pay_unit`는 유지(일회성·기존 UX 호환).

2. **Phase 2 — 유형별 집계**  
   - `market_benchmarks`에 `listing_type` 추가하거나, 유형별 benchmark 테이블/뷰 도입.  
   - refresh 로직을 유형별로 나누어: 소개는 소개비·수수료율, 매매는 월수금·매매가·조건별, 도급은 월도급금·조건별로 집계.

3. **Phase 3 — 노출**  
   - 상세/목록에서 유형별 “단가” 문구 + 유형별 시장 평균 비교(조건 매칭 시).  
   - 필요 시 “업계 평균 단가” 전용 페이지/필터 UX.

## 7. 요약 (다시 설명)

| 구분 | 설명 |
|------|------|
| **10% / 1~6배 / 80%** | 업계 암묵적 관행. **우리 평균의 기준이 아님.** 안내 문구로만 참고 가능. |
| **평균 단가의 출처** | **우리 시스템에 사용자가 올린 매매글·도급글·소개글의 실제 입력값만** 집계한 결과. |
| **평균 단가의 역할** | 사용자가 **글을 올릴 때** 수수료·금액을 **입력하는 순간** 그 유형·조건의 **현재 시장 평균**을 보여줘서, **시장 전체 흐름을 읽을 수 있게** 하는 장치. |
| **UX** | 글쓰기 화면에서 소개비/월수금/매매가/도급금 입력 필드 **옆 또는 아래**에 해당 지역·업무(·조건)의 평균 금액과 표본 수 노출. |

정리하면 “정기 소개는 10% 수수료”, “매매는 월 수금의 1~6배”, “도급은 월의 80%” 같은 업계 관행을 데이터로 쌓고, **실제 데이터를 바탕으로 평균 단가를 구성**하고, 그 평균을 **글 올릴 때 보여주는 장치**로 써서 시장 흐름을 읽을 수 있게 할 수 있다.
