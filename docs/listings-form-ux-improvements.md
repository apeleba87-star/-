# 현장거래 글쓰기 UX — 설계 개선점·보완점

사용자 제안 플로우(거래 유형 → 업무 종류 → 유형별 입력)를 기준으로, **개선점**과 **보완이 필요한 부분**을 정리한다.

---

## 0. 이 설계에 대한 의견

### 잘 맞는 부분

- **거래 유형 → 업무 종류** 2단계가 실제 업무(소개/매매/도급 × 정기/일회성)와 잘 대응한다. 도급은 정기만 보이게 하면 “일회성 도급” 혼동을 줄일 수 있어서 타당하다.
- **소개**: 예상금액 vs 금액 미정, 수수료 0~100%·+5% 버튼은 현장에서 “대략 몇 %로 갈지”만 정하는 경우를 잘 담는다. 예상 소개비(만원)까지 같이 보여주면 협상·설명이 수월하다.
- **평수·주횟수·난이도에 “현장 확인 필요”**는, 아직 조건이 안 정해진 소개/매매를 올릴 수 있게 해서 등록 진입 장벽을 낮춘다. 다만 “미정” 데이터는 **집계에서 빼는 것**이 좋다(평균 단가가 흐려지지 않도록).
- **정기 매매**: 월 수금·평수·주 횟수·업종·배수(0.1 단위)는 매매가 산출과 “평당 금액·평균 배수” 참고값 노출에 그대로 쓸 수 있어서, 현장에서 쓰는 용어와도 잘 맞는다.
- **도급**: 업종·평수·주 횟수·도급금액 + “이 조건의 평균 도급금”은 구인 측이 자신의 단가를 시장과 비교하기에 적절한 구조다.

### 유의할 점

- **필드 수**: 유형·업무별로 조합이 많아서, 한 화면에 다 넣으면 부담스러울 수 있다. “정기 소개 / 일회성 소개 / 정기 매매 / 일회성 매매 / 도급”별로 **보이는 블록만** 나오게 하면 단계가 분리되어 이해하기 쉽다(현재도 유형별로 블록이 나뉘어 있어서 방향은 맞음).
- **평균 단가·평균 배수**: “플랫폼 참고 평균”은 **플랫폼에 쌓인 실제 등록/성사 데이터만**으로 계산한다. 관행 안내(10%, 1~6배, 80% 등)는 **사용하지 않는다.** 플랫폼 참고 평균만 노출한다. 자체는 우리 집계만 노출하면 “우리 플랫폼 기준 참고값”이 명확해진다.
- **표본이 적을 때**: 업종·평수·주 횟수까지 맞춘 셀은 데이터가 적을 수 있으므로, **지역+업종 → 지역만** 같은 다단계 폴백과 “(이 지역 기준)” 같은 레이블이 있으면 신뢰도가 올라간다.

### 정리

- 설계 방향(소개/매매/도급, 정기/일회성, 예상금액·수수료율·평당·배수·평균 참고)은 **현장 거래 흐름과 목적에 잘 맞고**, 이미 적용한 “예상 수수료율·만원 표시”, “평당 월 수금”, “일회성 매매=예상매매가”, “다단계 폴백”과도 잘 이어진다.
- 남은 작업은 **정기 매매의 배수 입력·평균 배수 노출**, **도급의 업종·평수·주 횟수별 평균 도급금**, **평수/주횟수/난이도 “현장 확인 필요” 저장·집계 제외**를 단계적으로 넣는 것이다. 이 문서의 §1~§6 개선·보완과 §7 적용 사항을 따라가면 된다.

## 1. 전체 플로우 요약 (제안 기준)

| 1단계 거래 유형 | 2단계 업무 종류 | 비고 |
|-----------------|-----------------|------|
| 소개 | 정기 청소 / 일회성 청소 | 둘 다 선택 가능 |
| 매매 | 정기 청소 / 일회성 청소 | 둘 다 선택 가능 |
| 도급 | 정기 청소만 | 일회성 도급은 미지원 |

---

## 2. 소개 (정기청소·일회성 공통·차이)

### 2.1 제안 내용
- **예상금액**: 직접 입력 **또는** "금액 미정(직접 견적)" 선택
- **소개비 수수료**: 0~100% 직접 입력, **+5% 버튼**으로 5%씩 증가
- **평수**: 입력 + "현장 확인 필요" 옵션
- **주 횟수**(정기만): 입력 + "현장 확인 필요"
- **난이도**: 선택 + "현장 확인 필요"
- **기타 상세 내용**: 자유 입력

### 2.2 개선점

| 항목 | 개선 제안 | 이유 |
|------|-----------|------|
| **금액 미정 선택 시** | "금액 미정" 선택 시 예상금액 입력란 **비활성화** 또는 숨김, DB에는 `expected_amount = NULL` 등으로 저장 | 수수료율만 있으면 소개비 산출 불가 → "성사 시 협의" 문구 노출, 목록/검색에서 "금액 미정" 필터·배지 제공 시 일관성 |
| **수수료 0~100%** | 0·5·10…100% **프리셋 버튼**(5% 간격) + 직접 입력 병행 | +5%만 있으면 95%→100%에서 105%로 넘어갈 수 있음 → **상한 100%** 적용 및 프리셋으로 빠른 선택 |
| **+5% 버튼** | 100% 초과 시 **100%에서 멈춤** (더 이상 증가 불가) | 실수·과도 입력 방지 |
| **현장 확인 필요** | 평수/주횟수/난이도 각각 **체크박스 또는 토글**로 "현장 확인 필요" 선택 시, 해당 필드는 **집계·평균 산출에서 제외**하거나 별도 구간으로 분리 | 평균 단가 계산 시 "미정" 데이터가 섞이면 왜곡 가능 → 집계 정책 명확화 |
| **정기 vs 일회성** | 일회성 소개에는 **주 횟수 필드 없음** 유지 (제안대로) | 일회성은 주 단위 반복이 아니므로 적절함 |

### 2.3 보완 필요 사항

| 항목 | 보완 내용 |
|------|-----------|
| **소개비 "금액" 노출** | 수수료율만 입력 시, 예상금액이 있으면 "예상 소개비: 약 ○○만원" **자동 계산·미리보기** 표시 (예상금액 × 수수료율). 금액 미정이면 "성사 시 협의" 등 문구만 표시. |
| **저장 구조** | 소개비를 **수수료율(%)**로 저장할지, **산출 금액**도 함께 저장할지 정책 결정. 검색/필터는 "수수료율 구간"(예: 10% 이하) 또는 "금액 미정" 여부로 할 수 있음. |
| **평균 참고값** | "이 지역·업무·조건의 **평균 소개 수수료율**"을 등록 데이터 기반으로 계산해 노출할지 검토. 현재는 소개비(원) 중심이면, 수수료율 집계 컬럼 추가 시 위 플로우와 잘 맞음. |

## 3. 매매 (정기청소)

### 3.1 제안 내용
- 월 수금, 평수, 주 횟수, **업종** 입력
- **배수** 입력 (0.1 단위 조절)
- **평당 금액** 표시
- 업종·주 횟수에 맞춘 **평균 단가** 표시
- **배수**에 대해 업계 **평균 배수** 표시

### 3.2 개선점

| 항목 | 개선 제안 | 이유 |
|------|-----------|------|
| **평당 금액** | `월 수금 ÷ 평수`로 **자동 계산**하여 입력란 옆에 "평당 약 ○○원" 표시 (평수 0이면 "평수 입력 시 표시" 등) | 사용자가 직접 계산할 필요 없음, 오입력 검증에 활용 |
| **배수 0.1 단위** | 슬라이더 + 숫자 입력 병행 시, **최소 0.1·최대(예: 10~12)** 제한 및 단계값 0.1 고정 | 비현실적 배수(0.01, 99 등) 방지 |
| **매매가 자동 반영** | "매매가 = 월 수금 × 배수"로 **자동 계산** 후 미리보기 표시 (필요 시 매매가 직접 입력 모드도 선택 가능하게 할지 검토) | 배수만 정하면 매매가가 정해지므로, 입력 편의와 일관성 확보 |
| **업종** | 현재 "정기청소 종류"(사무실·계단·식당 등)와 **동일한 옵션**으로 두고, 매매·도급에서만 "업종" 라벨로 노출해도 됨. 정기 매매이므로 **정기청소 종류 = 업종**으로 통일 | 용어만 "업종"으로 바꾸면 됨, 카테고리 재사용 가능 |

### 3.3 보완 필요 사항

| 항목 | 보완 내용 |
|------|-----------|
| **평균 단가 산출 기준** | "업종·주 횟수에 맞춘 평균 단가"를 **평당 월 수금** 기준으로 할지, **매매가 기준**으로 할지 정의. 제안대로라면 "평당 금액(월 수금/평수)"과 "그 조건의 시장 평균 평당 금액"을 나란히 보여주는 것이 자연스러움. |
| **평균 배수** | "업계 평균 배수"는 **우리 플랫폼 등록 데이터**에서 `매매가/월수금` 비율을 집계한 값만 사용(기존 설계 원칙 유지). "정기청소 매매" 글의 월 수금·매매가가 있어야 하므로, 두 필드 모두 필수 또는 배수 입력 시 매매가 자동 계산 후 저장. |
| **일회성 매매** | "일회성 청소 매매"는 **월 수금·주 횟수·배수**가 어울리지 않을 수 있음. 일회성 매매 시에는 "예상 매매가" 1개 필드 + 평수·업종(일회성 종류)만 받고, "평균 단가"는 일회성 매매 데이터 기준으로 따로 집계하는 방안 검토. |

---

## 4. 도급 (정기청소만)

### 4.1 제안 내용
- 업종, 평수, 주 횟수, **도급금액**(월 도급금) 입력
- 업종·평수·주 횟수에 따른 **도급금액 평균 단가** 표시

### 4.2 개선점

| 항목 | 개선 제안 | 이유 |
|------|-----------|------|
| **평균 단가 단위** | "평당 월 도급금" 또는 "해당 조건(업종·평수·주횟수)의 월 도급금 평균"으로 통일. 표시 예: "이 조건 평균 약 ○○만원/월 (최근 N건)" | 사용자가 자신이 입력한 도급금이 시장 대비 어떤지 바로 이해 가능 |
| **업종·평수·주횟수** | 소개·매매와 동일하게 **정기청소 종류 = 업종**, 평수·주횟수는 숫자 + "현장 확인 필요" 옵션 가능하게 하면, 소개와 입력 패턴 통일 | 폼 일관성, 집계 시 "미정" 구간 분리 가능 |

### 4.3 보완 필요 사항

| 항목 | 보완 내용 |
|------|-----------|
| **집계 조건** | 평균 도급금을 **업종·평수대(구간)·주 횟수** 조합별로 내려줄 때, 표본이 적으면 상위 조건(업종만, 업종+지역 등)으로 폴백하는 다단계 노출 정책(기존 설계) 유지. |
| **일회성 도급** | 현재는 "도급 = 정기만"으로 두는 것이 제안과 같음. 향후 "일회성 도급" 수요가 있으면, 별도 유형으로 두고 "일회성 업종 + 예상 도급금" 구조로 확장 검토. |

## 5. 공통·전역 개선·보완

### 5.1 공통

| 항목 | 개선·보완 |
|------|-----------|
| **지역** | 거래 유형·업무 종류보다 **먼저** 두거나, 상단 고정. 평균 단가는 지역별로 달라지므로 선택 순서가 중요할 수 있음. |
| **필수/선택 표시** | 각 화면에서 필수 입력에 `*`, 선택 입력은 "선택" 또는 "(선택)" 라벨로 통일. |
| **저장 전 요약** | 제출 전 "입력 내용 요약" 한 줄 또는 카드로 보여주면, 금액·수수료율·배수 등 오입력 방지에 도움. |
| **목록/상세 노출** | "금액 미정", "현장 확인 필요", "배수 ○○배" 등이 목록·상세 페이지에서 어떻게 보일지(배지·아이콘·문구) 미리 정의. |

### 5.2 데이터·집계

| 항목 | 보완 내용 |
|------|-----------|
| **"현장 확인 필요"** | 평수/주횟수/난이도가 "미정"인 건은 **집계에서 제외**하거나, `unknown_*` 플래그로 저장 후 "조건 미정" 구간만 따로 집계. 그래야 "평균 단가"가 실제 입력된 값만으로 산출됨. |
| **수수료율 저장** | 소개비를 **율(%)**로 저장하면, 나중에 "예상금액 × 수수료율"로 소개비 금액을 복원 가능. 예상금액이 NULL이면 "금액 미정"으로만 활용. |
| **배수 저장** | 매매 시 **배수**와 **매매가** 둘 다 저장하거나, 배수만 저장하고 매매가 = 월수금×배수로 계산. 집계 시 "평균 배수"는 저장된 배수 또는 매매가/월수금으로 산출. |

### 5.3 접근성·UX

| 항목 | 개선 제안 |
|------|-----------|
| **진행 단계** | "거래 유형 → 업무 종류 → 금액·조건" 순서를 **스텝 인디케이터**로 보여주면, 현재 어디를 채우는지 명확해짐. |
| **기본값** | 기본 선택값은 비워두거나 "직접 입력" 유도. 관행 안내는 사용하지 않음. |
| **모바일** | 슬라이더·+5% 버튼·드롭다운이 작은 화면에서도 터치하기 쉽게 여백·크기 확보. |

## 6. 요약 표

| 구분 | 개선점 | 보완점 |
|------|--------|--------|
| **소개** | 금액 미정 시 입력 비활성화, 수수료 100% 상한·+5% 제한, 프리셋 버튼, "현장 확인 필요" 시 집계 제외 정책 | 예상 소개비 미리보기, 수수료율 vs 금액 저장 정책, 평균 수수료율 노출 여부 |
| **매매** | 평당 금액 자동 계산, 배수 범위·0.1 고정, 매매가 자동 계산 미리보기, 업종=정기청소 종류 통일 | 평균 단가·평균 배수 산출 기준, 일회성 매매 필드·집계 분리 |
| **도급** | 평균 단위(평당/조건별) 통일, 업종·평수·주횟수 입력 형태 통일 | 표본 적을 때 폴백, 일회성 도급 향후 확장 |
| **공통** | 지역 순서·필수/선택 표시·저장 전 요약·목록 노출 정의 | "현장 확인 필요" 집계 규칙, 수수료율·배수 저장 구조, 스텝·기본값·모바일 |

이 문서를 기준으로 필드 설계·DB 스키마·집계 함수를 단계적으로 맞추면, 제안하신 UX와 기존 "실제 데이터 기반 평균" 원칙을 함께 만족시킬 수 있다.

## 7. 적용된 사항 (구현 반영)

### 7.1 소개
- **예상 수수료율 + 약 ○○만원**: 예상금액·수수료율 입력 시 "예상 수수료율 ○○%, 약 ○○만원" 동시 표시. 금액 미정 선택 시 "(금액 미정)"만 표시.
- DB: `expected_amount`, `fee_rate_percent` 컬럼 추가(마이그레이션 029). 소개비 금액은 `expected_amount × fee_rate_percent / 100`으로 저장 가능.

### 7.2 매매
- **정기청소 매매**: 월 수금·평수 입력 시 **평당 월 수금** 자동 계산 표시("평당 월 수금: 약 ○○원"). 참고값은 등록 데이터 기준 중앙값 + 표본 적을 때 폴백 레벨 표시.
- **일회성 매매**: 유형 `sale_one_time` 추가. **예상매매가** 단일 필드로 입력. 목록/상세에서는 "예상매매가" 문구로 노출.

### 7.3 표본 적을 때 다단계 노출
- **getListingBenchmarks**에서 조회 순서: 1) 지역·유형·대분류·소분류 정확 일치 → 2) 소분류 제외(대분류만) → 3) 지역·유형만.
- 반환 행에 `fallback_level`: `"exact"` | `"main_category"` | `"region_only"`. 폼에서는 "업종 기준", "이 지역 기준" 등으로 표시.
- **별도 유형 확장**: 일회성 매매(`sale_one_time`), 향후 일회성 도급 등은 `listing_benchmarks`에 해당 `listing_type`·집계 로직을 추가한 뒤 동일 다단계 폴백을 적용하면 됨.

### 7.4 공통 (추천 방식)
- **지역 우선**: 현장 거래 유형 선택 직후 **지역(시·도/구)** 입력 배치.
- **필수/선택**: 라벨에 `*`(필수), "(선택)" 통일(일정, 매매가, 평수, 주 회수, 난이도, 상세 내용).
- **제출 전 요약**: "입력 내용 확인" 버튼으로 요약 토글. 지역·제목·유형별 금액(예상 수수료율·만원, 월 수금, 예상매매가, 월 도급금) 한 줄 요약 표시.
