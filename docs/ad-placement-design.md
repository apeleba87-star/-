# 자체 광고 슬롯 배치 설계 (관리자 직접 컨트롤)

우리가 직접 넣는 광고(구글 광고 아님)를 **어디에**, **어떤 방식으로** 둘지 정리한 문서입니다. 관리자는 `/admin/ads`(광고 슬롯)에서 지금처럼 슬롯별 캠페인을 관리합니다.

---

## 1. 현재 구조 (홈만 적용됨)

- **테이블**: `home_ad_slots`, `home_ad_campaigns`
- **슬롯**: `premium_banner`(Hero 직후), `native_card`(뉴스–현장공유 사이)
- **관리**: `/admin/ads`에서 슬롯 on/off, 슬롯별 캠페인(이미지·기간·CTA·sort_order) 등록·수정·삭제
- **노출**: 홈(`app/page.tsx`)에서 `getActiveHomeAds()`로 기간 내 캠페인 1건씩 조회 후 `AdPremiumBanner`, `AdNativeCard`로 렌더

---

## 2. “이곳에도” = 추가로 넣을 수 있는 위치

같은 방식(슬롯 + 캠페인)으로 확장할 수 있는 후보입니다.

| 위치 | 설명 | 슬롯 키 예시 | 비고 |
|------|------|----------------|------|
| **글 상세(포스트/리포트)** | `/posts/[id]` — 기사·일간 입찰 리포트 등 | `post_top`, `post_bottom` | 본문 위/아래 한 군데씩 권장. “이곳에도”에 가장 잘 맞음. |
| **아카이브(뉴스레터 회차)** | `/archive/[id]` | `archive_top`, `archive_bottom` | 회차 본문 위/아래 |
| **뉴스레터 이메일 본문** | 발송 시 HTML 내 삽입 | (기존 `ad_slots` 또는 별도) | 회차별/슬롯별 캠페인 조회 후 HTML에 삽입 |
| **목록 페이지** | `/listings`, `/jobs` 등 | `listings_sidebar`, `jobs_top` | 상단·사이드 한 슬롯씩 |

우선 적용하기 좋은 곳은 **글 상세(포스트/리포트)** 입니다.  
리포트·기사 조회 시에도 광고가 노출되도록 하려면 여기에 슬롯 1~2개(상단·하단) 두면 됩니다.

---

## 3. 어떤 식으로 할지 (방식)

### 3.1 슬롯 확장

- **옵션 A**: `home_ad_slots`에 키만 더 추가  
  - 예: `post_top`, `post_bottom` row 추가  
  - 장점: 기존 테이블·관리 화면 그대로 사용. `/admin/ads`에서 “글 상세 상단”, “글 상세 하단” 슬롯만 추가하면 됨.  
  - 단점: 테이블 이름이 “home”인데 글 상세 슬롯이 섞임(이름만 확장).

- **옵션 B**: 공용 `ad_slots` 테이블로 통합  
  - `ad_slots(key, name, enabled, placement)` 등으로 “홈”, “글상세”, “아카이브” 등 배치 구분.  
  - 캠페인 테이블은 `ad_slot_id`로 참조.  
  - 장점: 위치별로 확장·정리하기 좋음.  
  - 단점: 마이그레이션·기존 홈 로직 수정 필요.

**추천**: 단기에는 **옵션 A**로 `home_ad_slots`에 `post_top`, `post_bottom`만 추가해 빠르게 “글 상세에도” 노출. 나중에 슬롯이 더 늘어나면 옵션 B로 통합 검토.

### 3.2 캠페인 관리 (지금이랑 동일)

- 슬롯별로 캠페인 등록: 이미지, 제목, 설명, CTA 문구/URL, 노출 기간(start_date, end_date), sort_order.
- 기간 겹치면 `sort_order` 낮은 1건만 노출 (현재 홈과 동일).
- 관리자만 수정: `/admin/ads`에서 “글 상세 상단”, “글 상세 하단” 슬롯 선택 후 캠페인 CRUD.

### 3.3 노출 로직

- **글 상세**  
  - `/posts/[id]`에서 서버 컴포넌트로 “이 페이지용 슬롯”만 조회.  
  - 예: `getActiveAdsForPlacement('post_top')`, `getActiveAdsForPlacement('post_bottom')` 같은 식으로, 기존 `getActiveHomeAds()`와 같은 방식이되 슬롯 키만 다르게.  
  - 상단: 제목/날짜 아래, 본문 위. 하단: 본문 아래, 카드 닫는 부분 위.  
  - 컴포넌트는 홈의 `AdNativeCard` 재사용하거나, 같은 데이터로 작은 배너용 컴포넌트 하나 더 만들어도 됨.

- **아카이브/뉴스레터/목록**  
  - 같은 패턴: “배치(placement)” 또는 “슬롯 키”로 해당 페이지만 조회해서 1건씩 렌더.

---

## 4. 정리

- **어디에**:  
  - 1순위 — **글 상세** (`/posts/[id]`) 상단·하단.  
  - 이후 — 아카이브 회차, 뉴스레터 본문, 목록 페이지 등.
- **어떤 식으로**:  
  - 지금처럼 **슬롯 + 캠페인** 구조 유지.  
  - 슬롯만 `post_top` / `post_bottom` 등으로 늘리고, `/admin/ads`에서 동일하게 슬롯 on/off·캠페인 등록·기간 관리.  
  - 각 페이지에서는 “이 위치 슬롯”만 조회해서 기간 내 캠페인 1건 렌더.

이렇게 하면 “이곳(글 상세 등)에도 광고 자리”를 두면서, 관리자 모드 광고관리와 동일한 방식으로 직접 컨트롤할 수 있습니다.
